Necesito construir el backend completo de un sistema POS básico para una
evaluación académica.

OBJETIVO:
Desarrollar, probar, documentar y preparar para despliegue un backend serverless
que permita:
- Listar productos mediante GET /productos
- Registrar ventas mediante POST /ventas
- Usar AWS API Gateway como entrada HTTP
- Usar dos funciones AWS Lambda implementadas en Java 17
- Usar dos tablas DynamoDB: Productos y Ventas
- Usar AWS SAM mediante template.yaml
- Escribir pruebas unitarias con mocks para aislar DynamoDB
- Seguir estrictamente Spec-Driven Development

ENDPOINT GET /productos:
- Debe consultar la tabla DynamoDB Productos
- Respuesta exitosa:
  [{"id": "1", "nombre": "Laptop", "precio": 2500},
   {"id": "2", "nombre": "Mouse", "precio": 50}]
- Casos: respuesta exitosa, tabla vacía, error de conexión con DynamoDB

ENDPOINT POST /ventas:
- Debe registrar una venta en la tabla DynamoDB Ventas
- Request: {"productos": [{"id": "1", "cantidad": 1}]}
- Respuesta exitosa: {"message": "Venta registrada correctamente"}
- Casos: venta exitosa, body inválido, producto inexistente, error de DynamoDB

TEMPLATE.YAML (AWS SAM):
- API Gateway con GET /productos y POST /ventas
- Lambda ProductosFunction (Java 17)
- Lambda VentasFunction (Java 17)
- Tabla DynamoDB ProductosTable (clave primaria: id)
- Tabla DynamoDB VentasTable (clave primaria: id)
- Permisos IAM mínimos necesarios
- Variables de entorno para nombres de tablas

PRUEBAS UNITARIAS (JUnit 5 + Mockito):
- ProductosHandlerTest: éxito, tabla vacía, error DynamoDB
- VentasHandlerTest: éxito, body inválido, producto inexistente, error DynamoDB
- Aislar DynamoDB completamente con mocks
